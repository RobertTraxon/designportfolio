Problem

I was approached by the maths team at Twinkl and asked to design a product to help keep children engaged whilst learning and recalling their times tables.

Research

We ran a user-testing session with a group of children of varied ages in the office. This was to get insight into how they used our existing products, their main sticking points, and which external products they enjoyed. Here were some of the main questions I considered when gathering data for this project:

  1. Which maths games do they enjoy?
  2. Which Twinkl games do they enjoy?
  3. What do users dislike about the existing maths games?
  4. Identify the features that encourage longer play.

Around 35% of the test group preferred playing the games without background music.
Topmarks' Hit the Button game and TT Rockstars were the most popular maths games amongst the test group.
Younger children struggled with typing and generally preferred point & click-style games.
The group using games with scoring systems typically played for nearly twice as long.
Games with timers harmed the concentration of some of the children.

User Session
Another aspect was to create a game that kept the user playing for longer.
Solution

Create an interactive whack-a-mole style game template to help learners with times table recital.

Gamification

One of the major factors to make the game competitive and enjoyable was to bring simple gamification elements that would encourage users to keep playing. Through research, I found that a simple linear scoring system isnt always enough to engage users. Instead, I opted for a progressive scoring system that awards the user for accuracy as well as speed.

Accessibility

Another major aspect I wanted to focus on was making the game as accessible as possible. Some of the main takeaways that I wanted to incorporate based on the research were as follows:

  • Point and click only (No keyboard)
  • Optional timers
  • Optional scoring system
  • Individual audio controls

Prototype

With the main game mechanics agreed upon, these were the main focus points whilst prototyping:

  • Split subject areas into separate games
  • Split menu into categories and game types
  • Bold UI
  • Add accessibility settings menu
  • Multi-level scoring system
  • Non-timed game mode
  • Optional score system
  • Audio level sliders
  • Easy navigation
